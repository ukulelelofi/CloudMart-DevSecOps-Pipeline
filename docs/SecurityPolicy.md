# CloudMart Security Policy

## Objectives

- Secure software development
- Shift Left Security
- Continuous Compliance
- Automated Security Testing

## Security Controls

- Secret Scanning
- Static Application Security Testing (SAST)
- Dependency Scanning
- Container Security
- Infrastructure as Code Security
- Dynamic Application Security Testing (DAST)

## Release Policy

Critical findings:
- Block Release

High findings:
- Security Approval Required

Medium findings:
- Create Jira Ticket

Low findings:
- Fix in next sprint
